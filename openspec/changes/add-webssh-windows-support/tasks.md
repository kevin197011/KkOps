## 任务清单

### 1. 操作系统检测功能

- [ ] 1.1 实现 `detectOSType()` 函数
  - 执行 `uname -s` 检测 Linux/Unix
  - 执行 `ver` 或 `systeminfo` 检测 Windows
  - 解析命令输出判断系统类型
  - 返回系统类型（linux, windows, darwin, unknown）
  - 文件：`backend/internal/handler/webssh_handler.go`

- [ ] 1.2 在连接建立后调用检测
  - 在 SSH 会话创建后、PTY 请求前执行检测

- [ ] 1.3 添加系统类型到会话上下文

### 2. 终端类型适配

- [ ] 2.1 实现 `getTerminalTypeForOS()` 函数

  - 文件：`backend/internal/handler/webssh_handler.go`

- [ ] 2.2 修改 PTY 请求逻辑

- [ ] 2.3 测试终端类型兼容性

## 技术注意事项

1. **检测性能**: OS 检测会增加一次命令执行，约 100-500ms 延迟
2. **编码转换**: 编码转换有性能开销，但通常可接受
3. **缓存策略**: 可以缓存 OS 类型到主机信息，避免重复检测
4. **向后兼容**: 确保现有 Linux 连接不受影响
