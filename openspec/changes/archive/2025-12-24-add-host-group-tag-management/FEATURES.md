# 主机标签管理页面功能清单

## 概述
主机标签管理页面提供了完整的标签 CRUD 操作功能，允许管理员创建、编辑、删除和管理用于分类主机的标签。

## 前端功能

### 1. 标签列表展示
- ✅ **分页显示**: 支持分页浏览标签列表，默认每页 10 条
- ✅ **列表列信息**:
  - ID：标签的唯一标识
  - 标签：带颜色的标签预览（Tag 组件）
  - 名称：标签的名称
  - 颜色：颜色值显示（包含颜色块预览）
  - 描述：标签的描述信息（可省略显示）
  - 使用数量：显示该标签关联的主机数量
  - 创建时间：标签的创建时间
  - 操作：编辑和删除按钮

### 2. 标签创建
- ✅ **创建表单字段**:
  - 名称（必填）：标签名称，用于标识标签
  - 颜色：使用 ColorPicker 组件选择标签颜色，默认值 `#1890ff`
  - 描述（可选）：标签的详细描述信息
- ✅ **表单验证**: 名称字段必填验证
- ✅ **创建成功反馈**: 创建成功后显示成功提示并刷新列表

### 3. 标签编辑
- ✅ **编辑表单**: 与创建表单相同，包含名称、颜色、描述字段
- ✅ **数据回填**: 编辑时自动填充当前标签的数据
- ✅ **更新成功反馈**: 更新成功后显示成功提示并刷新列表

### 4. 标签删除
- ✅ **删除确认**: 使用 Popconfirm 组件进行删除确认，防止误操作
- ✅ **删除成功反馈**: 删除成功后显示成功提示并刷新列表

### 5. 标签使用情况
- ✅ **关联主机数量显示**: 在列表中显示每个标签关联的主机数量
- ✅ **数据关联**: 通过 `hosts` 关联字段显示使用该标签的主机列表

### 6. UI 特性
- ✅ **颜色预览**: 在列表中通过 Tag 组件显示带颜色的标签预览
- ✅ **颜色选择器**: 使用 Ant Design 的 ColorPicker 组件选择颜色
- ✅ **刷新功能**: 提供刷新按钮手动刷新标签列表
- ✅ **加载状态**: 显示数据加载状态
- ✅ **错误处理**: 显示操作失败的错误信息

## 后端 API

### 1. 标签 CRUD API
- ✅ **POST /api/v1/host-tags**: 创建新标签
  - 请求体: `{ name, color, description }`
  - 响应: `{ tag }`
  
- ✅ **GET /api/v1/host-tags**: 获取标签列表（支持分页和过滤）
  - 查询参数: `page`, `page_size`, `name` (可选过滤)
  - 响应: `{ tags, total, page, page_size }`
  
- ✅ **GET /api/v1/host-tags/:id**: 获取单个标签详情
  - 响应: `{ tag }`
  
- ✅ **PUT /api/v1/host-tags/:id**: 更新标签
  - 请求体: `{ name, color, description }`
  - 响应: `{ message }`
  
- ✅ **DELETE /api/v1/host-tags/:id**: 删除标签
  - 响应: `{ message }`

### 2. 主机标签关联 API
- ✅ **POST /api/v1/hosts/:id/tags**: 为主机添加标签
  - 请求体: `{ tag_id }`
  
- ✅ **DELETE /api/v1/hosts/:id/tags**: 移除主机的标签
  - 请求体: `{ tag_id }`

## 数据模型

### HostTag 模型
```go
type HostTag struct {
    ID          uint64
    Name        string  // 标签名称（唯一）
    Color       string  // 标签颜色（HEX格式，如 #1890ff）
    Description string  // 标签描述
    CreatedAt   time.Time
    // 关联
    Hosts       []Host  // 使用该标签的主机列表
}
```

## 集成功能

### 1. 主机管理页面集成
- ✅ **标签显示**: 在主机列表中显示每个主机关联的标签（带颜色）
- ✅ **标签选择**: 在主机编辑/创建表单中提供标签多选下拉框
- ✅ **标签过滤**: 在主机列表页面提供标签过滤功能
- ✅ **快速创建**: 在主机管理页面的标签选择器中提供"创建新标签"快捷入口

### 2. 路由和菜单
- ✅ **独立路由**: `/host-tags` 路由访问标签管理页面
- ✅ **菜单入口**: 可通过菜单或直接访问 URL 进入标签管理页面
