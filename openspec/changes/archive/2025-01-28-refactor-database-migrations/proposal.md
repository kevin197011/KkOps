# 数据库迁移系统重构提案

**创建日期**: 2025-01-28  
**状态**: ✅ 已完成  
**优先级**: 高

## 问题描述

当前项目中所有数据库表结构和索引的 SQL 语句都硬编码在 `backend/internal/models/database.go` 文件中，存在以下问题：

1. **代码臃肿**: 约 650 行硬编码 SQL 语句，代码可读性差
2. **维护困难**: 数据库结构变更需要修改 Go 代码，不利于版本控制
3. **迁移管理缺失**: 没有统一的迁移文件管理机制
4. **回滚困难**: 无法方便地回滚数据库变更

## 解决方案

将所有 SQL 变更语句统一迁移到 `backend/migrations/` 目录，实现基于文件的迁移系统：

1. **统一迁移文件管理**: 所有 SQL 变更语句统一到 `backend/migrations/` 目录
2. **基于文件的迁移系统**: 使用时间戳命名的 `.up.sql` 文件管理数据库结构变更
3. **自动迁移执行**: 应用启动时自动按时间戳顺序执行所有迁移文件
4. **代码清理**: 删除所有硬编码的建表和建索引 SQL 语句

## 预期收益

1. ✅ **代码简化**: 删除约 650 行硬编码 SQL 代码
2. ✅ **维护性提升**: 数据库结构变更通过迁移文件管理，便于版本控制
3. ✅ **迁移管理**: 支持时间戳排序、自动执行、回滚机制
4. ✅ **可追溯性**: 每个迁移文件都有明确的创建时间和描述
