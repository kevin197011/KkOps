## 📋 变更提案模板

创建新变更提案时，请在 `openspec/changes/` 目录下创建新文件夹，包含以下文件：

```
changes/
└── {change-name}/
    ├── proposal.md      # 变更提案（问题、解决方案、收益）
    ├── design.md        # 设计文档（架构、实现细节）
    ├── tasks.md         # 任务清单（实施步骤）
    └── specs/           # 规范文档（可选）
        └── {module}/
            └── spec.md
```

### 提案命名规范

- 使用小写字母和连字符：`add-feature-name`
- 使用动词开头：`add-`, `refactor-`, `remove-`, `enhance-`
- 描述清晰：能够从名称理解变更内容

---

## 🔄 变更提案生命周期

1. **提案阶段** - 创建 `proposal.md`，描述问题和解决方案
2. **设计阶段** - 创建 `design.md`，详细设计架构和实现
3. **实施阶段** - 创建 `tasks.md`，跟踪任务完成情况
4. **完成阶段** - 所有任务完成，更新状态为 ✅
5. **归档阶段** - 历史提案移至 `archive/` 目录
