# Change: 重构运维执行页面为快速执行操作台

## Why

当前的 `/executions` 页面（运维执行）虽然已经有任务创建和执行功能，但操作流程复杂，需要先创建任务再执行。用户需要一个**快速执行操作台**，可以：

- 直接选择任务模板或输入自定义脚本
- 快速选择目标主机（支持按项目/环境筛选）
- 立即执行，无需先保存任务
- 查看执行结果和日志
- 可选保存为任务供后续复用

## What Changes

### 前端变更

- **重构 `/executions` 页面**为执行操作台
  - 移除左侧工作流列表（已创建的任务管理移到 `/tasks`）
  - 改为单页表单式布局：模板选择 + 主机选择 + 执行按钮
  - 支持两种模式：
    - **模板模式**：从任务模板中选择，自动填充脚本内容
    - **自定义模式**：手动输入脚本内容和类型
  - 主机选择区域：
    - 支持按项目/环境筛选
    - 支持搜索主机名/IP
    - 支持批量选择（全选、按项目选、按环境选）
  - 执行选项：
    - 执行方式（同步/异步）
    - 超时时间设置
    - 是否保存为任务（可选）
  - 执行结果显示：
    - 显示执行状态和进度
    - 实时日志流式输出
    - 每个主机的执行结果单独展示

- **保留现有的执行历史查看功能**
  - 如果保存了任务，可以查看历史执行记录
  - 点击"查看历史"链接到任务的历史记录页面

### 后端变更

- 创建 `QuickExecute` API 端点（可选，或复用现有 API）
  - 支持不创建任务直接执行（临时执行）
  - 可选创建任务并执行（保存执行）

### UI/UX 改进

- **操作流程简化**：模板选择 → 主机选择 → 执行，三步即可完成
- **实时反馈**：执行开始后立即显示状态和日志
- **结果清晰**：每个主机的执行状态独立显示，便于对比

## Impact

- **受影响的页面**：`frontend/src/pages/executions/TaskManagementPage.tsx` - 完全重构
- **可能删除的组件**：
  - `WorkflowList` - 不再需要工作流列表
  - `ExecutionDetailsPanel` - 改为内联显示
  - `TaskEditModal` - 改为执行操作表单
- **保留的功能**：
  - 执行历史查看（如果保存了任务）
  - 执行记录详情和日志查看
- **API 兼容性**：复用现有 `/executions` 和 `/execution-records` API

## 与现有功能的关系

- **任务模板**（`/templates`）：作为执行脚本来源
- **任务执行**（`/tasks`）：保存的任务可以在任务执行页面管理
- **执行记录**：执行结果仍会保存到执行记录表，可以查看历史
