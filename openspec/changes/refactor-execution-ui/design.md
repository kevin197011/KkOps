# Design: GitHub Actions风格执行界面

## Context
批量操作和Formula部署是KkOps的核心功能，需要提供一致且直观的用户体验。参考GitHub Actions的工作流界面设计，提供清晰的执行状态展示和详细的日志查看功能。

## Goals
- 统一批量操作和Formula部署的界面风格
- 提供GitHub Actions风格的工作流运行列表
- 支持点击查看详细执行日志
- 实时显示执行进度和状态

## Non-Goals
- 不改变后端API接口
- 不改变数据模型结构
- 不添加新的业务功能

## Decisions

### 1. 组件架构
采用共享组件模式，创建以下可复用组件：

```
components/
├── ExecutionLogViewer.tsx    # 执行日志详情弹窗
├── WorkflowRunList.tsx       # 工作流运行列表
└── WorkflowRunItem.tsx       # 单个运行项
```

### 2. 界面布局
采用两栏布局：
- 左侧：配置区域（主机选择、命令/Formula配置）
- 右侧：执行区域（执行按钮、当前执行状态）
- 底部：历史记录列表（GitHub Actions风格）

### 3. 执行记录列表设计
参考GitHub Actions的工作流运行列表：
```
┌─────────────────────────────────────────────────────────────┐
│ ● 执行名称                                    状态    耗时   │
│   #ID · 命令/Formula · 开始时间                              │
├─────────────────────────────────────────────────────────────┤
│ ✓ base_motd - 2025/12/26 14:40:16           成功    5s     │
│   #385 · Formula部署 · 2分钟前                               │
├─────────────────────────────────────────────────────────────┤
│ ✗ test.ping - 2025/12/26 14:35:00           失败    2s     │
│   #123 · 批量操作 · 7分钟前                                  │
└─────────────────────────────────────────────────────────────┘
```

### 4. 日志查看器设计
采用终端风格的日志显示：
- 深色背景 (#1e1e1e)
- 等宽字体 (Monaco, Menlo, Consolas)
- 彩色日志级别标识
- 支持全屏模式
- 支持日志复制和下载

### 5. 状态颜色规范
- pending: 黄色 (#faad14)
- running: 蓝色 (#1890ff)
- completed/success: 绿色 (#52c41a)
- failed: 红色 (#ff4d4f)
- cancelled: 灰色 (#8c8c8c)

## Risks / Trade-offs
- 风险：大量日志可能影响性能
- 缓解：限制日志显示行数，支持分页加载

## Migration Plan
1. 创建新组件
2. 逐步替换现有页面组件
3. 保持API兼容性
4. 测试验证后部署
